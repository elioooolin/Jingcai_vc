# 图片压缩功能说明

## 问题背景

云函数调用时数据大小限制为 **5120 KB (5 MB)**，当上传的图片 base64 编码后超过这个限制会导致调用失败，报错：
```
errCode: -401002 api parameter error | errMsg: callFunction data size must be less than 5120 KB
```

## 解决方案

### 1. 自动图片压缩

选择图片时自动进行智能压缩：

- **尺寸压缩**: 限制图片最大边为 1500px，保持原始宽高比
- **质量压缩**: 压缩质量设为 60%，平衡清晰度和文件大小
- **优先压缩**: `wx.chooseMedia` 使用 `sizeType: ['compressed']` 优先获取压缩图

### 2. 裁剪后二次压缩

用户裁剪图片后，自动进行二次压缩，确保裁剪后的图片也符合大小要求。

### 3. 提交前大小检查

在将图片转为 base64 后，检查大小：
- 如果超过 **4.5 MB**（留有安全余量），拒绝提交
- 提示用户裁剪图片或重新选择更小的图片

## 技术实现

### 压缩策略

```typescript
// 1. 计算目标尺寸（保持比例）
const maxSize = 1500; // 最大边限制
if (width > maxSize || height > maxSize) {
  按比例缩放到 maxSize
}

// 2. 使用 wx.compressImage 压缩
wx.compressImage({
  src: filePath,
  quality: 60,           // 压缩质量
  compressedWidth: targetWidth,
  compressedHeight: targetHeight
})
```

### 兼容性

- ✅ **iOS**: 支持相机拍照和相册选择
- ✅ **Android**: 支持相机拍照和相册选择
- ✅ **自动降级**: 压缩失败时自动使用原图（可能会因大小限制导致提交失败）

### 用户体验

1. **选择图片时**:
   - 显示 "处理图片中..." 加载提示
   - 完成后提示 "图片已优化"

2. **裁剪图片时**:
   - 裁剪后自动压缩
   - 显示 "处理图片中..." 和 "图片已优化"

3. **提交时**:
   - 显示图片 base64 大小（控制台）
   - 如果超过限制，清晰提示用户

## 文件大小估算

### Base64 编码后的大小
- Base64 会增加约 33% 的大小
- 实际大小 = base64字符串长度 × 0.75 / 1024 (KB)

### 压缩效果示例
- **原图**: 4000×3000, 约 8 MB
- **压缩后**: 1500×1125, 约 500-800 KB (取决于内容)
- **Base64后**: 约 700-1100 KB

## 最佳实践

### 用户操作建议
1. 优先使用相机直接拍照（通常尺寸更合适）
2. 从相册选择时，选择较小的图片
3. 使用裁剪功能去除不必要的部分
4. 如果提示图片过大，重新选择或裁剪

### 开发建议
- 监控控制台日志，查看压缩前后的尺寸
- 根据实际使用情况调整压缩参数
- 考虑在服务端进一步优化图片存储

## 错误处理

### 压缩失败
- 自动使用原图
- 可能在提交时因大小超限而失败

### 提交失败
- 显示图片大小和限制
- 提示用户裁剪或重新选择

## 后续优化建议

1. **渐进式压缩**: 如果第一次压缩后仍超限，自动进行更强的压缩
2. **分片上传**: 对于大图片，考虑分片上传到云存储
3. **直接上传**: 考虑使用 `wx.cloud.uploadFile` 直接上传，避免 base64 转换
4. **CDN 优化**: 使用图片 CDN 服务，自动优化和缓存

## 相关 API

- `wx.chooseMedia`: 选择图片/视频
- `wx.getImageInfo`: 获取图片信息
- `wx.compressImage`: 压缩图片
- `wx.editImage`: 编辑/裁剪图片（可选）
