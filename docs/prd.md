# 微信小程序需求文档

## 一、项目概述
### 1. 小程序名称
爱睦月子餐

### 2. 目标用户群体
- 客户：需要进行月子餐点餐的产妇。
- 管理员：负责管理客户信息和点餐的营养师。

## 二、主要功能
### 1. 用户角色管理
- 用户分为客户和管理员两种角色。
- 支持微信绑定一键登录功能，以简化用户注册和登录流程。

### 2. 客户账号管理
- 管理员能够创建客户角色账号，需输入客户信息：
  - 绑定手机号、姓名、生日、入住日期、入住天数、入住门店、入住房号、饮食偏好、高补餐次数。

### 3. 点餐功能
- 客户可以选择相应日期进行点餐。
- 每日可选择的餐品：
  - 早餐：主菜二选一（必选）。
  - 午餐：主菜四选二（必选）、汤品二选一（必选）。
  - 晚餐：同午餐选项。
  - 高补餐：四选一（可选）。

### 4. 菜品展示
- 点菜页面每个菜品都显示菜名和图片，用户点击图片可进入菜品详情页，以了解更多信息。

### 5. 点餐确认及管理
- 客户提交点餐后，需由管理员进行确认。
- 管理员可以手动替换客户所点菜品，以满足客户需求或特殊情况。

### 6. 数据导出功能
- 管理员能够以表格形式导出指定日期范围内所有客户的确认已点菜品，以便后厨备餐。

### 7. 点餐历史与未来菜单状态
- 客户可以查看自己的所有点餐历史，显示过去的所有订单，包括所点菜品、下单时间等信息。
- 客户能查看未来已点菜单的状态，状态包括：
  - **待提交**：选择了菜品但未完成下单。
  - **待确认**：已提交订单，等待管理员确认。
  - **已确认**：订单已被管理员确认，进入后续处理流程。

## 三、用户操作流程
### 1. 客户操作流程
1. 通过微信扫描二维码或搜索进入小程序。
2. 微信一键登录或注册成为客户。
3. 填写相关客户信息。
4. 选择日期，进入点餐页面。
5. 按照每日菜品选择要求进行点餐。
6. 备注栏填写特殊需求，如少盐、不吃蒜等（optional）。
7. 提交订单，等待管理员确认。
8. 访问“我的点餐历史”页面查看点餐历史及未来菜单状态。

### 2. 管理员操作流程
1. 通过微信一键登录进入后台管理系统。
2. 创建和管理客户账号信息。
3. 查看指定日期内的客户点餐情况。
4. 确认或修改客户所点菜品，必要时手动替换。
5. 导出确认已点菜品的报表，以便备餐。

## 四、设计要求
- 页面风格应简洁、清新，符合月子餐饮的健康、舒适的主题。
- 优先使用易于辨别的图标和清晰的字体，确保用户体验流畅。

## 五、非功能需求
- 系统应能支持高并发用户访问，确保性能稳定。
- 数据存储应采用安全、可靠的方式，确保用户信息和订单数据的安全性。
- 界面应具备良好的响应速度及兼容性，适配不同手机型号。
